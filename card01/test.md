# 《穗并町：再婚家庭》角色卡介绍

## 简介

这是一个设定在2010年日本关东近郊小镇"穗并町"的再婚家庭题材角色卡。

故事背景：user与带着两个女儿的寡妇水谷美咲再婚半年。家庭成员包括：完全顺从的妻子美咲、渴望被关注的小女儿千夏、保持距离的大女儿雪乃（有男友）。

核心设定：这不是简单的家庭模拟卡，而是一个带有**动态堕落系统**的复杂机制卡。通过内置的JavaScript脚本，角色会根据玩家的行为真实地改变，从抗拒到接受，从正常到堕落，整个过程有数值支撑、有行为特征、有具体示例。

世界观完整度：包含详细的小镇地理、学校设定、商业区、自然地貌、工作日/周末时间线、家庭内部场景等，为剧情提供真实的生活土壤。

## 玩法说明

### 基础玩法

1. **日常互动**：根据时间线（工作日/周末）与家庭成员互动
2. **场景切换**：在家中各个房间、小镇各个地点之间移动
3. **关系推进**：通过对话和行为影响角色状态

### 核心机制：性刺激接受度系统

角色卡内置了一个复杂的数值计算系统，追踪五种行为：
- 阴道性交
- 肛交
- 口交吞精
- 公共场所性行为
- 羞辱或暴力

每次行为会记录刺激强度（0.1-2.0），系统自动计算四种接受度：
- **背德**：对禁忌行为的接受程度
- **应激情绪**：对暴力/恐惧的性唤起
- **责任转移**：对服从/支配的接受
- **被欲望验证**：对被侵犯/被需要的渴望

### 辐射机制

独特设定：对一个角色的调教会"辐射"影响其他角色（设定为血脉感应）。

例如：频繁与母亲发生性行为，会逐渐影响两个女儿的接受度，即使她们没有直接参与。不同角色的辐射效率和接受效率不同：
- 母亲：辐射效率0.15，对"责任转移"接受效率高（2倍）
- 妹妹：辐射效率0.2，对"被欲望验证"接受效率高（2倍）
- 姐姐：辐射效率0.25，对"背德"接受效率高（2倍）

### 动态演化

角色不是静态的。随着接受度提升，她们的：
- **程度描述**会改变：完全抗拒→轻微接受→中度接受→高度接受→极度接受→成瘾状态
- **行为特征**会改变：从推开逃离到主动配合再到依赖寻求
- **具体示例**会更新：系统提供当前状态下的典型行为示例

AI会根据这些数据调整角色的反应和对话，实现真实的堕落过程。

## 卡片特色

### 1. 高度系统化

不依赖AI的随机发挥，而是用数值和逻辑驱动角色变化。每个行为都有明确的影响，每个状态都有具体的表现。

### 2. 三位一体的性格设计

每个角色都有"性格调色盘"设计：
- 母亲：顺从（底色）+ 母爱（主色调）+ 畏惧（点缀）
- 妹妹：渴望被看见（底色）+ 讨好（主色调）+ 聪明（点缀）
- 姐姐：完美主义（底色）+ 理性克制（主色调）+ 背德感受度（点缀）

不是单一标签，而是多种性格衍生组合，形成立体的人物。

### 3. 三面性机制

每个角色有多个"面"，根据情境切换：
- 姐姐有"完美的学生""温柔的女友""背德的自己"三面
- 不同面之间有过渡、有渗透、有冲突
- 堕落不是变成另一个人，而是某一面逐渐占据主导

### 4. 真实的世界观

不是抽象的"某个家庭"，而是具体的2010年穗并町：
- 有车站前的商店街、国道旁的超市、镇后山的杂木林
- 有工作日和周末的不同时间线
- 有昭和遗风与平成日常的交界感
- 天气和温度会根据日期自动生成

### 5. 技术实现

使用Zod schema和JavaScript实现：
- 自动计算接受度
- 自动生成行为特征
- 自动更新示例库
- 只读字段防止AI手动修改
- JSON Patch格式更新变量

## 注意事项

### 使用前提

1. **平台要求**：需要支持JavaScript脚本的AI平台（如SillyTavern + 酒馆助手）
2. **模型要求**：建议使用理解能力强的大模型，能准确解读数值和状态
3. **成人内容**：本卡包含大量成人向、禁忌向内容，仅供成年人使用

### 使用建议

1. **循序渐进**：不要一开始就做极端行为，让接受度自然提升
2. **关注数值**：定期查看角色的接受度状态，了解当前进度
3. **利用辐射**：先调教母亲，利用辐射机制影响女儿们
4. **尊重设定**：姐姐有男友，这是她堕落过程的重要元素，不要轻易删除
5. **时间线配合**：根据工作日/周末的不同时间安排行动

### 常见问题

**Q：可以修改辐射效率吗？**
A：可以。在脚本中找到`获取辐射效率`和`获取辐射接受效率`函数，修改数值即可。

**Q：能添加新的行为类型吗？**
A：可以。在`调教影响权重`中添加新类型，并在`调教记录`schema中添加对应字段。

### 技术说明

- 变量更新使用JSON Patch格式，AI需要在回复末尾输出更新指令
- 以`$`开头的字段是只写的（如`$调教记录`），以`_`开头的字段是只读的（如`_性刺激接受度`）
- 天气和温度基于日期哈希生成，同一天永远是同样的天气
- 刺激强度基础值0.1-2.0，多人在场时有倍率加成
